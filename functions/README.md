# Functions Library

Persistent implementation guidance for Sara's Portfolio. Read the relevant function file before changing a page, component, list view, detail view, modal, image treatment, navigation, or shared visual pattern.

## Library Rules

- Function files document current implementation and intended reuse rules.
- When a known gap, defect, or future improvement is added to any function file, add a matching item to `docs/tasks.md`.
- When a matching task is completed, update the relevant function file with the outcome.
- Prefer existing components and documented patterns before creating new markup or CSS.
- Do not patch one card, image, modal, or page in isolation when the issue belongs to a shared pattern.
